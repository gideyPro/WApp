# Fix KYC Status Bug

## Problem
The Settings → KYC Verification screen shows a form even when user has already submitted KYC and it's pending approval.

## Observations
- **Settings KYC menu item**: Shows "Pending" correctly ✅
- **Settings → KYC Verification Screen**: Shows Form instead of Pending ❌  
- **Profile modal from Home**: Shows "Pending" correctly ✅

## Root Cause
**File:** `lib/data/services/kyc_service.dart` line 25

```dart
// Current (WRONG):
status: verification['verified'] != true ? (data['status'] ?? 'none') : 'approved',

// Should be:
status: verification['verified'] != true ? (verification['status'] ?? 'none') : 'approved',
```

## API Response
```json
{
  "success": true,
  "data": {
    "verified": false,
    "has_submission": true,
    "verification": {
      "status": "pending",  // ← Status is HERE in verification object
      "document_type": "national_id"
    }
  }
}
```

## Why It Fails
| Code Reads | What Exists | Result |
|------------|-------------|--------|
| `data['status']` | null (doesn't exist) | Defaults to "none" |
| Screen shows | Form (status = "none") | ❌ |

## Fix Implementation

### File: `lib/data/services/kyc_service.dart`

**Line 25** - Change from:
```dart
status: verification['verified'] != true ? (data['status'] ?? 'none') : 'approved',
```

**To:**
```dart
status: verification['verified'] != true ? (verification['status'] ?? 'none') : 'approved',
```

This correctly parses `verification.status` ("pending") instead of looking for non-existent `data.status`.
